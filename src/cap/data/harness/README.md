The `.harness/` directory contains MCP governance configuration for a CAP workspace. The `mcp-policy.json` file defines the default-deny security posture: which operations are allowed, dangerous command patterns to flag, budget limits, and audit settings. During `cap init`, these defaults are copied into the workspace root at `.harness/` where they can be customized per-project. The `manifest.json` is auto-generated and tracks SHA-256 hashes of security-relevant files for drift detection.
